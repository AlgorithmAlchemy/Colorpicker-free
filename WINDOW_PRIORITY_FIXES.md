# Исправления для работы окна поверх игр

## Проблема
Программа не работала поверх всех окон, когда включена игра или другие приложения с высоким приоритетом.

## Внесенные изменения

### 1. Улучшенные флаги окна
- Добавлен `Qt.X11BypassWindowManagerHint` для обхода оконного менеджера
- Добавлен `Qt.WindowTransparentForInput` для прозрачности ввода
- Добавлен атрибут `Qt.WA_AlwaysStackOnTop` для принудительного отображения поверх всех окон

### 2. Агрессивное восстановление окна
- Уменьшен интервал проверки видимости с 2 секунд до 1 секунды
- Добавлен новый таймер для агрессивного восстановления каждые 500мс
- Множественные попытки поднятия окна с разными интервалами (50мс, 100мс, 200мс, 300мс, 500мс)

### 3. Улучшенный Windows API
- Принудительная установка окна поверх всех через `win32gui.SetWindowPos`
- Дополнительные проверки Z-order окна
- Повторные попытки установки через Windows API

### 4. Новые методы
- `_aggressive_window_restore()` - агрессивное восстановление для игр
- `_verify_windows_topmost()` - проверка и принудительная установка через Windows API
- `_force_windows_topmost_after_toggle()` - принудительная установка после переключения режима

### 5. Улучшенное переключение режима "поверх всех"
- Автоматическое применение Windows API при включении режима
- Более агрессивные флаги окна
- Дополнительные попытки поднятия окна

## Результат
Теперь программа должна работать поверх всех окон, включая игры и приложения с высоким приоритетом, благодаря:
- Максимально агрессивным флагам окна
- Постоянной проверке и восстановлению видимости
- Использованию Windows API для принудительного поднятия
- Множественным попыткам восстановления с разными интервалами
